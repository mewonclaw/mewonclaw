Governance Model

How decisions
get made.

We are building a platform owned by its community. Here is how that works.

Principles

Transparent

All decisions, finances, and code are public.

Community-led

Active users vote on direction.

Trust-minimized

Code over promises. Verifiable, not just stated.

Decision Process

  1. 01
    Proposal

    Anyone can submit via GitHub or Discord.

  2. 02
    Discussion

    Two weeks of community feedback.

  3. 03
    Vote

    One week voting period. Active builders only.

  4. 04
    Implementation

    If passed, on the roadmap. Tracked publicly.

Voting Rights

One active builder = one vote. To qualify as active you need:

  • At least 1 published AI build
  • Account active in the last 90 days
  • No abuse violations

We deliberately do NOT use token-weighted voting. Building > buying.

What gets voted on?

Yes — votable
  • Feature priorities
  • Policy changes
  • Treasury allocation
  • Partnerships
No — fixed
  • Core staying free
  • User data ownership
  • Open code policy
  • Privacy commitments
